============================================================================
                    CodeNearby API & Pricing System Summary
============================================================================

OVERVIEW
--------
CodeNearby provides AI-powered developer search and GitHub analysis APIs 
with a token-based pricing system. The API helps developers find other 
developers, analyze profiles, and discover repositories using natural 
language queries.

============================================================================
                           API ENDPOINTS
============================================================================

1. AI-Connect Developer Search
   Endpoint: POST /api/v1/developers
   Token Cost: Variable (200-800 tokens per request, average: 400) [consts/pricing]
   Description: Find developers using natural language queries
   
   Input: { "query": "Find React Native developers in New York" }
   Output: List of matching developers with GitHub profiles
   
   Features:
   - Natural language processing
   - Location-based filtering
   - Skills/technology matching
   - Experience level detection

2. AI-Connect Profile Analysis  
   Endpoint: POST /api/v1/profile or /api/v1/profile/analyze
   Token Cost: Variable (300-2000 tokens per request, average: 1000) [consts/pricing]
   Description: Get AI analysis of GitHub developer profiles
   
   Input: { "username": "octocat" }
   Output: Detailed profile analysis with insights
   
   Features:
   - Expertise assessment
   - Activity level analysis
   - Project impact evaluation
   - Professional background insights

3. AI-Connect Repository Search
   Endpoint: POST /api/v1/repositories
   Token Cost: Variable (150-1200 tokens per request, average: 600) [consts/pricing]
   Description: Find repositories using natural language
   
   Input: { "query": "React UI component libraries" }
   Output: Matching repositories with detailed metadata
   
   Features:
   - Technology-based search
   - Project type classification
   - Popularity filtering
   - Feature matching

4. User Tier Information
   Endpoint: GET /api/v1/users/tier
   Token Cost: 0 tokens (free) [consts/pricing]
   Description: Get current user tier and token balance

============================================================================
                           AUTHENTICATION
============================================================================

All API requests require authentication using API keys:

1. Include API key in request headers:
   Header: x-api-key: your_api_key_here

2. API keys are tied to user accounts and tier levels

3. Requests perform a pre-check using an estimated cost (from consts/pricing), then consume tokens based on actual AI usage when available.

4. Requests return current token balance and usage information

============================================================================
                           HOW IT WORKS
============================================================================

ACCOUNT CREATION & TIERS
-------------------------
1. Users start with FREE tier (1,000 daily tokens)
3. First token purchase upgrades to PREMIUM tier
4. Tiers unlock additional features and higher limits

TOKEN CONSUMPTION
-----------------
1. API requests validate key without consuming tokens
2. AI processing generates response and calculates actual tokens used
3. System checks if user has sufficient tokens
4. If sufficient, tokens are consumed from balance
5. If insufficient, request returns 402 error

TOKEN BALANCE SYSTEM
--------------------
- Daily Free Tokens: Reset every 24 hours to tier limit
- Purchased Tokens: Never expire, consumed after daily tokens
- Total Balance: Daily + Purchased tokens
- Consumption Order: Daily tokens first, then purchased tokens

REAL-TIME USAGE TRACKING
-------------------------
- Tokens consumed based on actual AI model usage
- Variable pricing based on query complexity
- No pre-estimation or fixed costs per endpoint
- Detailed usage analytics and reporting
